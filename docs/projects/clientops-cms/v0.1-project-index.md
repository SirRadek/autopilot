# ClientOps CMS v0.1 Project Index

Date: 2026-06-15
Status: v0.1-local-baseline-map
Project package: `autopilot-clientops-cms`
Runtime: Next.js App Router, Payload CMS 3, PostgreSQL 16 through Docker Compose

## Purpose

ClientOps CMS v0.1 is a local-first operations backend for Radeq client work. It stores canonical client, site, lead, workflow, and opportunity data. It also contains a project-specific Decision Mesh that governs what automation may do with that data.

This document is the project map for further work. When there is ambiguity, use this order:

1. Local code and tests
2. This project index
3. `docs/projects/clientops-cms/architecture.md`
4. Project mesh documents under `docs/projects/clientops-cms/decision-mesh/`
5. Older design specs under `docs/superpowers/specs/`

The former Autopilot control-plane v0.2.0 tree is preserved under `archive/autopilot-control-plane-v0.2.0/` as migration source material. It is intentionally outside the active build and typecheck surface.

## Core Versus Mesh

The core is the running application and canonical data model:

- Payload CMS admin and API
- PostgreSQL-backed collections
- Public lead intake
- Workflow tasks
- Workflow events
- Opportunity records
- Local scripts and verification gates

The mesh is the governance layer:

- Stop conditions
- Source-of-truth rules
- Auth and token boundaries
- Retry/manual override policy
- Advisory model rules
- Opportunity source and retention rules

The mesh does not replace the database, the CMS, or the APIs. It explains how automation is allowed to interpret and act on them.

## What Not To Build In v0.1

- Production deployment
- Fully autonomous workers
- Automated outreach to opportunity requesters
- AI/model output as canonical CMS state
- Runtime provider usage-limit enforcement
- Broad live crawling
- Login/paywall scraping
- Hlidac Statu live runs
- Payload GraphQL integrations
- Parallel external worker pools until scoped tokens, atomic claim behavior, and supervision are verified against the target runtime
- Email/SMS delivery
- Polished public client website
