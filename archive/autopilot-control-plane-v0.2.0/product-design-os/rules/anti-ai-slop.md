# Anti AI Slop

Avoid default AI-generated design habits unless the project explicitly justifies
them.

## Common Risks

- repeated equal card grids
- palette-only variants
- generic SaaS gradients
- dark neon AI look by default
- fake dashboard hero
- bento grid as a reflex
- vague CTA copy
- decorative motion without user value
- every version sharing the same layout skeleton
- stock-like imagery that does not explain the offer
- copying a reference site's structure closely enough that only text and colors
  changed
- treating named variants as one reusable component with different tokens
- mascot, demo, or motion layers that do not change the user's understanding of
  the offer
- mixing every attractive mood into one incoherent style

## Required Checks

- Does the visual language communicate what the product or service does before
  reading every paragraph?
- Is there one strong visual anchor, not just a color change?
- Does each section answer a distinct user question?
- Is motion explaining, guiding, or confirming something?
- Which theme axes are being crossed, and what is the clarity or SEO
  counterweight?
- Is the source an approved commercial-safe asset, or only an inspiration
  reference with a clean-room brief?
- Would the same layout still make sense if the brand text were swapped with a
  random SaaS product? If yes, redesign the concept.

## Accepted Direction Pattern

For marketing, portfolio, and creative-service previews, a good A/B/C/D set
means separate compositions with distinct jobs, not a single shell:

- `Guided Offer Map`: conversion-safe explanation path from buyer problem to
  demo or request.
- `Cat Concierge`: playful guide where mascot behavior helps attention,
  navigation, or trust without hiding content.
- `Studio Proof`: editorial proof surface with deliverables, handoff evidence,
  demos, and checks.
- `Demo Worlds`: demo-first hub where each example is a different miniature
  product world.

Each accepted direction must own its first viewport composition, proof rhythm,
CTA logic, motion concept, mobile behavior, and fallback plan.
