# Change Requests

Every new request during a project must be classified before implementation.

## Classification

- A: clarification of original goal
- B: scope expansion
- C: direction change
- D: backlog idea
- E: conflict with goal

## Template

```json
{
  "date": "YYYY-MM-DD",
  "request": "",
  "classification": "A|B|C|D|E",
  "reason": [],
  "recommendation": "",
  "requires_scope_change": false,
  "owner_decision": ""
}
```
