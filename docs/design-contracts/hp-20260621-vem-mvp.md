## Decisions Already Made

- Primitive = layout-box graph with source binding, NOT a pixel raster.
- Overlap rule: hit-test returns the full stack; resolution picks the topmost
  PAINT-VISIBLE, semantically-meaningful node; invisible `pointer-events` overlays
  are demoted (reconciles Codex topmost-z + Gemini visible-wins).
- Pixel grid is at most a coarse human-pointing index, never the authority.
- Identity: `siteId = sha1(domPath)` deterministic; `instanceKey` appends a stable
  framework key/text digest when available; never raw DOM index alone.

## Expected Output

1. `capture-element-map.ts` exporting:
   - `captureElementMap(input): Promise<ElementMap>` — reuses/extends the chromium
     harness; per viewport, in-page `evaluate()` walks visible elements collecting
     for each: `id`, `siteId`, `instanceKey`, `parentId`, `domPath`, `role`,
     `accessibleName`, `textSnippet`, `rect{x,y,w,h}`, `zPath`, `depth`, `visible`,
     `hitTestable`, `sourceRef{file,line,col,component}|"unknown"`, `styleDigest`
     (incl. spacing: padding/margin/gap + font/color), `stateFlags`,
     `identityConfidence`. Also a screenshot path per viewport.
   - `resolvePointToPassport(map, viewport, x, y): Passport | null` — PURE function,
     offline hit-test over captured boxes (filter containing point → topmost
     visible/meaningful via zPath then depth; demote zero-paint overlays).
   - A CLI: `--url <u> | --html-file <f> [--output-dir <d>] [--xy <vp> <x> <y>]`.
     With `--xy` it prints the resolved passport (echo payload: thumbnail-crop
     path, role:name, id, source ref).
2. `element-map.schema.json` describing `{ vemVersion, url, captured_at, viewports:
   [{ name, width, height, screenshot_path, nodes: [Passport] }] }`.
