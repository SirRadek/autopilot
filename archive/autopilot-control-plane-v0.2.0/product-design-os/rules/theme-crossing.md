# Theme Crossing

Use theme crossing for creative marketing, portfolio, agency, and product-demo
work when the goal is memorability without losing conversion clarity.

## Theme Axes

Available mood and visual axes:

- lightness, freedom, space, ease
- idea, novelty, distinctiveness, standing out
- modern retro, pixel vibe, soft nostalgia
- art, artistic direction, written/editorial craft
- futurism, modeled 3D, animated systems
- chill, calm, focused, calculated, minimal
- dopamine, colorful, high-energy contrast

Do not use every axis at once. Pick:

1. one primary personality axis,
2. one contrast axis,
3. one product-proof axis.

Example:

```text
Cat Concierge = playful mascot + calm trust + proof-led demos
Studio Proof = bold agency art + calculated minimalism + handoff evidence
Demo Worlds = dopamine color + pixel/future hint + static demo links
```

## SEO And Conversion Compromise

Every crossed theme must survive these compromises:

- The offer, H1, service explanation, proof, and CTA stay in static readable
  HTML.
- Visual assets support the message; they do not replace product/service copy.
- Motion and 3D are progressive enhancements with reduced-motion and mobile
  fallback.
- Titles, descriptions, headings, canonical, robots, sitemap, internal links,
  and indexability remain explicit.
- Core Web Vitals, mobile layout, contrast, focus states, and text overlap are
  checked before approval.
- Dopamine color or pixel/futurist style must not weaken trust, readability, or
  qualified request flow.
- Minimal/calm style must not become generic, empty, or interchangeable.

This is a minimum floor, not a demand for perfect SEO on every page. Use
`rules/design-seo-tradeoff.md` to choose whether the direction is `seo_led`,
`balanced`, `brand_led`, or `experimental_showcase`.

For brand-led and experimental work, memorability, motion, art direction, and
shareability may outweigh exhaustive keyword sections or perfect scores. The
tradeoff is acceptable when the page still has a static readable purpose, normal
links/CTA, accessible controls, mobile fallback, reduced-motion behavior, and an
explicit indexability decision.

## Direction Output

For each direction, state:

- chosen theme axes,
- tradeoff profile,
- why that crossing fits the target user,
- what the opposing force is for clarity or trust,
- what content remains static for SEO,
- what motion/asset layer is optional,
- what gets rejected because it harms performance, accessibility, or conversion.
